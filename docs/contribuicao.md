# Contribuição

## Objetivo

Esta seção descreve o fluxo de contribuição adotado pela equipe do projeto ProtectKids.

O objetivo é manter a organização do código, padronização das entregas e facilitar a colaboração entre os integrantes.

---

## Fluxo de Trabalho

O desenvolvimento do projeto segue o seguinte fluxo:

1. Criação de uma branch para a funcionalidade
2. Desenvolvimento e testes locais
3. Commit das alterações
4. Abertura de Pull Request
5. Revisão do código
6. Integração na branch principal

---

## Padrão de Branches

| Tipo | Exemplo |
|---|---|
| Feature | `feature/dashboard` |
| Correção | `fix/classificacao` |
| Documentação | `docs/mkdocs` |

---

## Padrão de Commits

Os commits devem ser descritivos e objetivos.

### Exemplos
